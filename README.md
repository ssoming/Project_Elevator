## 2. 주요 기능

- **논블로킹 층간 이동 제어**: 버튼으로 목표 층을 지정하면 `for`문 없이 시간 비교 기반으로 스텝모터·LED바·FND를 동시에 구동
- **포토인터럽트 기반 정밀 층 감지**: 층별 센서로 현재 위치를 실시간 파악해 목표 층 도착을 판정하고 깜빡임으로 알림
- **발판 온습도 안내**: 엘리베이터 앞 발판 감지 시 DHT11 값을 LCD에 자동 출력

---

## 3. 담당 역할

**포토인터럽트 기반 발판 감지 및 온습도 안내 시스템 구현, 엘리베이터 상태머신 공동 설계**

- **발판 센서 + LCD/DHT11 온습도 출력 (`photo.c`)**
  포토인터럽트 센서 상태를 폴링 방식으로 판단해 발판 진입/이탈을 감지하고, 감지 시 DHT11 값을 읽어 LCD에 온습도 정보를 출력하는 로직 구현
- **엘리베이터 상태머신 (`elevator.c`) — 팀원과 공동 구현**
  논블로킹 FSM(IDLE / MOVING_UP / MOVING_DOWN / ARRIVED_BLINK) 설계, 버튼 입력 및 모터·LED바·FND 제어 로직

### 핵심 구현

**① 논블로킹 FSM 기반 다중 동작 제어**

`HAL_GetTick()` 기반 시간 비교로 모터 이동(2ms 주기)과 LED바 애니메이션(100ms 주기)을 블로킹 없이 동시 수행

**② 포토인터럽트 폴링 기반 발판 상태 감지**

EXTI는 즉시 반응용 플래그로만 소비하고, 실제 상태 전환은 폴링으로 판단해 신호 유실에도 안정적으로 LCD ON/OFF 처리

**③ 센서 타겟팅 기반 도착 판정**

회전량 대신 층별 포토인터럽트 센서 감지 시점을 기준으로 정지 위치를 판단, 도착 시 LED바 깜빡임으로 알림

## 5. 트러블슈팅

| 발생 문제 | 발생 원인 | 해결 방안 | 결과 |
|-----------|-----------|-----------|------|
| Blocking Code로 인한 시스템 멈춤 | 한 동작 수행 중 다른 동작으로 전환되지 않음. `for`문 기반 제어로 시스템이 멈춘 것처럼 동작 | 논블로킹(NonBlocking) 구조로 변경 | 여러 기능 동시 수행 가능 |
| if–elseif 조건 충돌 | 층 비교 조건이 동시에 참이 되어 초기 상태에서 FND가 3층으로 잘못 표시 | 각 층 비교 조건을 독립적인 `if`문으로 분리 | 현재 층 판별 정상 동작 |
| Stepper 회전량 불명확 | 이동에 필요한 정확한 회전량을 알 수 없어 목표 위치 제어 기준이 불명확 | 포토인터럽트 센서를 기준으로 감지 시점까지 회전 후 정지하도록 타겟팅 | 정확한 층 위치에서 정지 |

## 6. 디렉토리 구조 (Directory Structure)

| 파일 | 역할 |
|------|------|
| `main.c` | 시스템 초기화 및 메인 루프(`Elevator_Loop`, `photoLoop` 호출) |
| `elevator.c` | 엘리베이터 상태머신, 모터/LED바/FND 제어 |
| `photo.c` | 발판 포토인터럽트 감지 및 LCD 온습도 출력 |
| `dht11.c` | DHT11 온습도 센서 통신 드라이버 |
| `I2C_LCD.c` | I2C 기반 LCD 제어 드라이버 |
| `Project02_Team5.ioc` | STM32CubeMX 프로젝트 설정 파일 |
